# Support

Use [GitHub Issues](https://github.com/Ishikawa-Hidekazu/public-source-extractor/issues) for reproducible bugs, documentation gaps, and focused feature requests after the repository is public.

Before opening an issue:

1. Confirm Python 3.11 or newer is in use.
2. Run the latest source version.
3. Remove credentials, tokens, signed query parameters, private URLs, and private output.
4. Include the command shape, exit code, operating system, Python version, and redacted error JSON.

Provider availability, anonymous limits, and third-party page extraction quality are not guaranteed. A Firecrawl Keyless `429` or temporary provider failure does not by itself indicate a CLI defect.

For security reports, follow [SECURITY.md](SECURITY.md) instead of opening a public issue.
